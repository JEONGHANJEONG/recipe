# 요리 레시피 관리 웹 서비스

## 1. 프로젝트 제목

**요리 레시피 관리 웹 서비스**

---

## 2. 주제 설명

이 프로젝트는 사용자가 자신만의 요리 레시피를 등록하고 관리할 수 있는 웹 서비스이다.
회원가입과 로그인을 통해 개인 레시피를 작성할 수 있으며, 다른 사용자의 레시피는 열람하고 댓글, 추천, 신고 기능을 사용할 수 있다.
관리자는 전체 레시피, 댓글, 신고 내역을 확인하고 관리할 수 있다.

---

## 3. 주요 기능

### 사용자 기능

* 회원가입 및 로그인 / 로그아웃
* 전체 레시피 목록 조회
* 레시피 상세 보기
* 레시피 검색 기능
* 카테고리, 난이도, 조리시간 필터 기능
* 보유 재료 기준 70% 이상 일치 레시피 필터 기능
* 레시피 등록, 수정, 삭제
* 본인이 작성한 레시피만 수정 및 삭제 가능
* 조리 방법에 사진 첨부 가능
* 댓글 작성, 수정, 삭제
* 추천하기 및 추천 취소
* 레시피 신고 기능

### 관리자 기능

* 전체 회원 목록 조회
* 전체 레시피 조회, 수정, 삭제
* 전체 댓글 조회 및 삭제
* 신고 목록 조회
* 신고 상태 변경

    * 접수
    * 처리중
    * 처리완료

---

## 4. CRUD 기능 설명

| 구분     | 기능                                    |
| ------ | ------------------------------------- |
| Create | 회원가입, 레시피 등록, 댓글 작성, 추천 등록, 신고 작성     |
| Read   | 레시피 목록 조회, 상세 조회, 내 레시피 조회, 관리자 목록 조회 |
| Update | 레시피 수정, 댓글 수정, 신고 상태 변경               |
| Delete | 레시피 삭제, 댓글 삭제, 추천 취소                  |

## 5. DB 구조

### users 테이블

| 컬럼명        | 설명                    |
| ---------- | --------------------- |
| id         | 회원 고유 번호              |
| username   | 사용자 이름                |
| email      | 로그인 이메일               |
| password   | 암호화된 비밀번호             |
| role       | 사용자 권한, user 또는 admin |
| created_at | 가입일                   |

### recipes 테이블

| 컬럼명          | 설명         |
| ------------ | ---------- |
| id           | 레시피 고유 번호  |
| title        | 요리 이름      |
| category     | 카테고리       |
| ingredients  | 재료         |
| instructions | 조리 방법      |
| cooking_time | 조리 시간      |
| difficulty   | 난이도        |
| created_at   | 등록일        |
| user_id      | 레시피 작성자 번호 |

### comments 테이블

| 컬럼명        | 설명            |
| ---------- | ------------- |
| id         | 댓글 고유 번호      |
| recipe_id  | 댓글이 달린 레시피 번호 |
| user_id    | 댓글 작성자 번호     |
| content    | 댓글 내용         |
| created_at | 작성일           |
| updated_at | 수정일           |

### recipe_recommendations 테이블

| 컬럼명        | 설명         |
| ---------- | ---------- |
| id         | 추천 고유 번호   |
| recipe_id  | 추천한 레시피 번호 |
| user_id    | 추천한 사용자 번호 |
| created_at | 추천한 날짜     |

### recipe_images 테이블

| 컬럼명            | 설명              |
| -------------- | --------------- |
| id             | 이미지 고유 번호       |
| recipe_id      | 이미지가 첨부된 레시피 번호 |
| image_filename | 저장된 이미지 파일명     |
| created_at     | 이미지 등록일         |

### recipe_reports 테이블

| 컬럼명         | 설명         |
| ----------- | ---------- |
| id          | 신고 고유 번호   |
| recipe_id   | 신고된 레시피 번호 |
| reporter_id | 신고한 사용자 번호 |
| reason      | 신고 사유      |
| detail      | 신고 상세 내용   |
| status      | 신고 처리 상태   |
| created_at  | 신고일        |
| handled_at  | 처리일        |


## 6. 프로젝트 특징

이 웹 서비스는 기본적인 CRUD 기능뿐만 아니라 로그인 기반 권한 관리, 댓글, 추천, 신고, 관리자 페이지 기능을 포함한다.
또한 SQL Injection 방지를 위해 파라미터 바인딩을 사용하고, 입력값 검증을 통해 검색어, 회원가입 정보, 댓글, 신고 내용 등의 사용자 입력을 안전하게 처리한다.
